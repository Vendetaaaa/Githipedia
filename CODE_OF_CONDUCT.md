## The Short Version

Don't make a community, make a home :)

---

## What We Expect

**Be respectful**
Disagree with someone's idea, not with them as a person. Everyone here is volunteering their time & effort for the greater good.

**Be welcoming**
New contributors are learning. Experienced ones make mistakes. Both deserve patience and a helping hand, not frustration.

**Be constructive**
If something is wrong, explain why and suggest a fix. A comment like "this is incorrect, the actual behavior is X because Y" is useful. 

> Just "this is wrong" is not a solution.

**Be honest**
If you do not know something, say so. If a guide you wrote is outdated, flag it. Accuracy matters.

**Give credit**
If you are building on someone else's work or idea, acknowledge it.

---

## What Is Not Okay

These behaviors will lead to a warning or removal from the project, depending on severity:

- Insults, personal attacks, or hostile language toward any contributor;
- Harassment of any kind, in issues, pull requests, discussions, or anywhere else connected to this repo;
- Dismissing questions, especially from beginners;
- Deliberately spreading wrong information;
- Spam, self-promotion unrelated to Githipedia's purpose, or off-topic noise;
- Sharing someone else's private information without their permission;
- Programs having malicious content of any kind;

---

## Scope

This code of conduct applies to everything inside this repository: issues, pull requests, commit messages, discussions, and any communication that happens in the context of contributing to Githipedia.

---

## Reporting a Problem

If someone is violating this code of conduct, or if you have been on the receiving end of behavior that should not be happening here, you can:

- **Open a private issue** and mark it clearly as a conduct report
- **Contact the maintainer directly** through GitHub and/or email provided ( at the moment @Vendetaaaa )

Reports will be handled with discretion. You will not be ignored, and the situation will be looked into.

---

## What Happens After a Report

Depending on what happened:

1. A private message to the person involved, explaining what was wrong and what needs to change
2. A public warning on the relevant issue or pull request
3. Temporary or permanent removal from the project

The goal is always to fix the situation and keep the community healthy, not to punish people for the sake of it.

## A Note on GitHub Itself

Since Githipedia is entirely focused on GitHub as a platform, a lot of the work here involves documenting how GitHub works and how to use it well. Please keep in mind that GitHub also has its own [Community Guidelines](https://docs.github.com/en/site-policy/github-terms/github-community-guidelines) and [Terms of Service](https://docs.github.com/en/site-policy/github-terms/github-terms-of-service). Everything in this repo should align with those, and nothing here should be used to help someone misuse or abuse GitHub's systems.

If you think something is missing here or could be stated more clearly, open an issue. This code of conduct belongs to the community, and the community can improve it.
